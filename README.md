1. **SRP**  
Принцип соблюдается. В User и Teacher поля и конструктор которые являются неотъемлемой часть любого пользователя и учителя - ФИО.
2. **OCP**  
Принцип соблюдается. Есть общий класс User в котором обозначенны обязательные поля для любого наследника - ФИО. В Teacher добавляет своё поле Id. Другой наследник может добавить свои поля, например, возраст или телефон.
3. **LSP**  
Явно не прослеживается. В классах User и Teacher есть только методы toString, которые переопределются у наследников с необходимыми, конкретно для них, полями.
4. **ISP**  
Принцип соблюдается. В интерфейсе UserController два метода create и sendOnConsole разнесенеы на два интерфейсе, так как не обхательно наслединкам понадобиться реализовывать оба метода.
5. **DIP**  
Не прослеживается.
